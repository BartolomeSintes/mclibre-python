Diferencias entre Python 2 y Python 3

Estas son algunas diferencias entre Python 2 y Python 3 relevantes para los ejercicios planteados en estos apuntes.

  • En Python 2 los nombres de las variables no podían contener caracteres no ingleses, como acentos, eñes, c cedillas, etc. En Python 3 sí.
    • Python 2: 
      >>> año = 1492
SyntaxError: invalid syntax
  • En Python 2 print era una palabra reservada. En Python 3 print() es una función.
  • En Python 2 una coma al final de una sentencia print omitía el salto de línea final. En Python 3, hace falta indicar el caracter final print( ..., end="")
  • En Python 2 si en un programa las cadenas de texto contenían caracteres no ingleses, el programa debe comenzar por una directiva que le indique el juego de caracteres. En Python 3 no hace falta.
  • En Python 2 la función de entrada de datos era la función raw_input() . En Python 3 es la función input() .
  • En Python 2, los enteros había dos tipos de enteros, enteros normales (con 32 bits de precisión) y enteros largos (de precisión ilimitada). En Python 3 sólo hay un tipo de enteros, de precisión ilimitada.
  • En Python 2 la divisón de enteros daba un resultado entero (si da un decimal, se truncaba al entero inferior). En Python 3 la solución es decimal.
    • Python 2: 
      >>> distancia = 100
>>> tiempo = 4
>>> distancia / tiempo
25

      >>> variable = 100
>>> variable * 60 / 60
100
>>> variable / 60 * 60
60

      Para entender por qué se obtienen resultados distintos, hay que tener en cuenta que Python efectúa los cálculos de izquierda a derecha y en cada paso el resultado es un número entero:

      • En el primer caso primero multiplica (100 * 60 = 6000) y luego divide (6000 / 60 = 100)
      • En el segundo caso primero divide (100 / 60 = 1 al tratarse de números enteros) y después multiplica (1 x 60 = 60).
    • Python 3: 
      >>> distancia = 100
>>> tiempo = 4
>>> distancia / tiempo
25.0

      >>> variable = 100
>>> variable * 60 / 60
100.0
>>> variable / 60 * 60
100.0

Módulos

No hay que guardar nunca un programa con el nombre de un módulo, porque si en ese programa (o en otro del mismo directorio) se importa el módulo, Python no importará el módulo sino el propio programa.

Excepciones

while True:
    try:
        numero = int(input("Dígame un número entero: "))
        break
    except ValueError:
        print("No ha escrito un número entero")
print(numero)
