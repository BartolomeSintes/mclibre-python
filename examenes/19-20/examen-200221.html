Ejercicio 1

MPTC

192011

Para indicar el tiempo transcurrido desde una fecha pasada, los mayas utilizaban la llamada cuenta larga o serie inicial, en la que se cuenta en kins, uinals, tuns, katúns y baktúns. Un kin es un día, un uinal son 20 kins, un tun son 18 uinals, un katún son 20 tuns y un baktún son 20 katúns.

Escriba un programa que solicite una cantidad de días y que responda la cuenta larga maya correspondiente.

CUENTA LARGA MAYA
Escriba una cantidad de días: -100
Por favor, no escriba números negativos.

CUENTA LARGA MAYA
Escriba una cantidad de días: 365
365 días son 0 baktún, 0 katún, 1 tun, 0 uinal y 5 kin.

CUENTA LARGA MAYA
Escriba una cantidad de días: 36524
36524 días son 0 baktún, 5 katún, 1 tun, 8 uinal y 4 kin.

CUENTA LARGA MAYA
Escriba una cantidad de días: 3652425
3652425 días son 25 baktún, 7 katún, 5 tun, 11 uinal y 5 kin.

Ejercicio 3

MPTC

192013

Escriba un programa que pida un código HSL de color y devuelva el código RGB correspondiente.

El procedimiento de cálculo (fuente: Wikipedia) para convertir los valores de matiz (H), saturación (S) y luminosidad (L) en rojo (R), verde (G) y azul (B) es el siguiente:

  • Cálculo de c (croma):
    • u es el valor absoluto de (2 multiplicado por L menos 1)
    • v es 1 menos u
    • c es v multiplicado por S
  • h2 es H dividido por 60
  • Cálculo de x:
    • y es (el resto de h2 dividido por 2), menos 1
    • z es 1 menos el valor absoluto de y
    • x es c multiplicado por z
  • h3 es h2 redondeado a número entero hacia arriba
  • Cálculo de r1, g1, b1:
    • Si h3 es 1, r1 es c, g1 es x y b1 es 0
    • Si h3 es 2, r1 es x, g1 es c y b1 es 0
    • Si h3 es 3, r1 es 0, g1 es c y b1 es x
    • Si h3 es 4, r1 es 0, g1 es x y b1 es c
    • Si h3 es 5, r1 es x, g1 es 0 y b1 es c
    • Si h3 es 6, r1 es c, g1 es 0 y b1 es x
    • En otro caso, r1, g1 y b1 son 0
  • m es L menos (c dividido por 2)
  • r es r1 más m
  • g es g1 más m
  • b es b1 más m
  • Por último, multiplique r, g y b por 255 y redondee el resultado al entero más próximo para obtener R, G y B.
CONVERTIDOR HSL a RGB
Introduzca las componentes HSL:
Matiz (entero de 0 a 360) H = 400
Saturación (decimal de 0 a 1) S = 0.5
Luminosidad (decimal de 0 a 1) L = 0.5
Matiz incorrecto. Inténtelo de nuevo.

CONVERTIDOR HSL a RGB
Introduzca las componentes HSL:
Matiz (entero de 0 a 360) H = 271
Saturación (decimal de 0 a 1) S = 0.759
Luminosidad (decimal de 0 a 1) L = 0.527
El color es R = 137, G = 43, B = 226

CONVERTIDOR HSL a RGB
Introduzca las componentes HSL:
Matiz (entero de 0 a 360) H = 82
Saturación (decimal de 0 a 1) S = 0.39
Luminosidad (decimal de 0 a 1) L = 0.302
El color es R = 85, G = 107, B = 47

CONVERTIDOR HSL a RGB
Introduzca las componentes HSL:
Matiz (entero de 0 a 360) H = 195
Saturación (decimal de 0 a 1) S = 0.533
Luminosidad (decimal de 0 a 1) H = 0.79
El color es R = 173, G = 216, B = 230

Ejercicio 4

MPTC

192014

Escriba un programa que simule un juego de dados entre dos jugadores (Alejandro y Beatriz). Cada jugador lanza dos dados. Si no coincide ningún dado, gana Alejandro. Si coincide uno de los dados, gana Beatriz. Si coinciden los dos dados, empatan.

Notas:

  • Este juego no es un juego justo, ya que Beatriz tiene un poco de ventaja (en las 1296 partidas posibles, Alejandro ganaría en 600, Beatriz en 630 y empatarían en 66).
  • Los ejemplos siguientes no cubren todas las situaciones posibles.
JUEGO DE DADOS
Alejandro ha sacado 3 y 5.
Beatriz ha sacado 1 y 3.
Ha ganado Beatriz.

JUEGO DE DADOS
Alejandro ha sacado 5 y 7.
Beatriz ha sacado 6 y 1.
Ha ganado Alejandro.

JUEGO DE DADOS
Alejandro ha sacado 3 y 3.
Beatriz ha sacado 3 y 1.
Ha ganado Beatriz.

JUEGO DE DADOS
Alejandro ha sacado 5 y 2.
Beatriz ha sacado 2 y 5.
Han empatado.
